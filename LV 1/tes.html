Latihan Menyiapkan Development Environment

Penting!

Pada latihan ini, Anda akan membuat AWS resources berikut:

  • AWS Cloud9 environment
  • Amazon Simple Storage Service (Amazon S3) bucket
  • Amazon API Gateway REST API
  • Amazon DynamoDB table
  • AWS Step Functions state machine
  • AWS Lambda functions

Semua layanan di atas dicakup oleh AWS Free Tier. Silakan kunjungi halaman tersebut, pelajari, dan pastikan Anda tidak melebihi batas AWS Free Tier yang sudah ditentukan.

Di akhir kelas, Anda wajib menghapus semua resources yang telah dibuat di kelas ini guna menghindari billing (penagihan) di waktu mendatang.

Di production environment, Anda wajib mengamankan AWS account root user, membuat IAM user baru dan mengasosiasikannya dengan customer managed policy, serta menggunakan IAM user baru tersebut untuk aktivitas sehari-hari sesuai kebutuhan. Customer managed policy memberikan kontrol yang lebih presisi ketimbang AWS managed policy. Dengan customer managed policy, Anda bisa membuat kebijakan (policy) sendiri dengan menentukan aktivitas apa saja yang diperbolehkan untuk AWS resource tertentu. 

Kelas ini tak akan membahas tentang IAM. Anda bisa belajar lebih lengkap tentang AWS IAM di kelas Architecting on AWS (Membangun Arsitektur Cloud di AWS) atau membaca AWS documentation berikut ini.

Berhubung ini adalah lingkungan yang bersifat latihan dan Anda menggunakan akun AWS untuk keperluan pribadi, langkah-langkah tersebut tidak akan kita praktikkan. Jadi, kita tetap menggunakan AWS account root user. Namun, bila Anda ingin menerapkan best practice tersebut, tentu sangat diperbolehkan dan itu hal yang bagus.

Untuk saat ini, kita masih belum membahas CI Pipeline di AWS karena akan fokus ke tahapan Code di alur CI/CD terlebih dahulu.

Tujuan dari latihan ini adalah membiasakan diri Anda dengan layanan AWS Cloud9 dan AWS CodeCommit (yang sudah kita pelajari di materi sebelumnya), sekaligus berkenalan lebih lanjut dengan Trivia App (tanpa CI Pipeline) serta mempelajari source code dan mencoba berinteraksi langsung dengannya.

Agar mendapatkan gambaran yang lebih jelas, berikut adalah tahapan proses untuk latihan kali ini.

  1. Kita akan mulai dengan membuat AWS Cloud9 environment untuk mempersiapkan development environment Anda. Di environment tersebut, lakukan clone source code dari Dicoding Academy GitHub repository yang akan digunakan dalam latihan ini.
  2. Selanjutnya, kita akan men-deploy infrastruktur backend dari aplikasi Trivia App menggunakan AWS Serverless Application Model Command Line Interface (AWS SAM CLI). Anda akan menggunakan AWS SAM untuk membuat semua AWS resources yang meng-hosting backend dari Trivia App, seperti Amazon API Gateway gateway, AWS Lambda functions, Amazon DynamoDB table, dan AWS Step Functions state machine. 
  3. Setelah itu, menguji coba frontend terlebih dahulu di AWS Cloud9.
  4. Kemudian, Anda akan men-deploy frontend dari aplikasi Trivia App, yakni aplikasi web React yang di-host di Amazon S3 bucket. 
  5. Setelah aplikasi aktif dan berjalan, taruhlah source code aplikasi Trivia App di source control menggunakan AWS CodeCommit.

Kini, Anda sudah memiliki gambaran besar untuk latihan ini dan paham apa saja yang akan kita lakukan. Selanjutnya, mari kita praktikkan.


Membuat AWS Cloud9 Environment dan Mengunduh Aplikasi Trivia App

Pertama, kita perlu membuat AWS Cloud9 environment sebagai development environment. Setelah itu, clone aplikasi Trivia App dari AWS Cloud9. 

  1. Silakan login ke AWS Management Console. Pilihlah Region yang diinginkan, disarankan untuk memilih Region Singapore karena paling dekat dengan Indonesia dan memiliki dukungan layanan yang cukup lengkap (dibandingkan Jakarta).
    Catatan: Jika Anda kesulitan dalam mengubah region, silakan tetap menggunakan region default.

  2. Masuk ke halaman AWS Cloud9 melalui kolom pencarian. Pastikan Anda masih berada di Region Singapore (ap-southeast-1). 

  3. Klik tombol Create environment. 

  4. Untuk Name, isikan trivia-app, sedangkan untuk Description, silakan isi sesuai keinginan Anda. Lalu, pilih Next step. 

  5. Pertahankan konfigurasi dari Environment settings sesuai default. Pilih Next step.

  6. Simak konfigurasinya baik-baik. Intinya, kita akan menggunakan EC2 environment (direct access) dengan spesifikasi t2.micro (free tier) yang memiliki sistem operasi Amazon Linux 2. Jika dirasa aman, lanjut pilih Create environment. 

  7. Tunggu beberapa menit, AWS Cloud9 sedang membuatkan environment untuk Anda.

  8. Pada AWS Cloud9 terminal (bagian bawah), clone aplikasi Trivia App dengan menjalankan perintah berikut. 
    1. git clone -b trivia-app https://github.com/dicodingacademy/a428-cicd-labs.git

Sampai di sini, Anda sudah berhasil membuat AWS Cloud9 environment dan mengunduh aplikasi Trivia App.


Deploy Infrastruktur Backend

Di tahapan ini, Anda akan deploy berbagai infrastruktur backend dari aplikasi Trivia App menggunakan AWS SAM CLI.

  1. Pada AWS Cloud9 terminal, ubah direktori ke folder a428-cicd-labs dan gunakan AWS SAM untuk build file template.yaml. 
    1. cd a428-cicd-labs/
    2. sam build
    3. sam deploy --guided

  1. Kemudian, Anda akan disuguhkan dengan AWS SAM deploy configuration. Untuk Stack Name, ketikkan trivia-app. Untuk konfigurasi lainnya, biarkan sesuai default dengan menekan Enter.

    1. Stack Name [sam-app]: trivia-app
    2. AWS Region [ap-southeast-1]: <Tekan Enter>
    3. #Shows you resources changes to be deployed and require a 'Y' to initiate deploy
    4. Confirm changes before deploy [y/N]: <Tekan Enter>
    5. #SAM needs permission to be able to create roles to connect to the resources in your template
    6. Allow SAM CLI IAM role creation [Y/n]: <Tekan Enter>
    7. Save arguments to configuration file [Y/n]: <Tekan Enter>
    8. SAM configuration file [samconfig.toml]: <Tekan Enter>
    9. SAM configuration environment [default]: <Tekan Enter>

  1. Setelah menunggu proses deploy, seharusnya Anda akan mendapatkan konfirmasi bahwa infrastruktur backend sukses dibuat.

    1. Successfully created/updated stack - trivia-app in ap-southeast-1

  1. Masih di terminal, perhatikan pada bagian tabel Outputs. Salin Value dari API Gateway websockets endpoint. Nilainya akan tampak seperti berikut:

    1. wss://xxxxxxxxxx.execute-api.ap-southeast-1.amazonaws.com/Prod

Oke, kita telah berhasil deploy infrastruktur backend. Selanjutnya, mari kita urus bagian frontend.


Menguji Coba Frontend di AWS Cloud9

Pada tahapan ini, Anda akan menginstal Node package manager (NPM) dependencies. Anda juga akan menilik preview dari aplikasi Trivia App di AWS Cloud9 environment.

  1. Silakan buka berkas a428-cicd-labs/front-end-react/src/config.js (Anda bisa memanfaatkan toggletree alias Explorer yang ada di sebelah kiri), kemudian perbarui nilai WebsocketEndpoint dengan endpoint yang tadi Anda salin. Jika sudah, pastikan Anda menyimpan (save) berkas tersebut dengan menekan kombinasi CTRL+S. 
    1. module.exports = {
    2.   WebsocketEndpoint: 'wss://xxxxxxxxxx.execute-api.ap-southeast-1.amazonaws.com/Prod'
    3. };

  1. Kembali ke AWS Cloud9 terminal, atur versi Node ke v16 (codename: Gallium).

    1. nvm install lts/gallium
    2. nvm alias default lts/gallium

  1. Pindah ke folder front-end-react, instal NPM dependencies, lalu jalankan Trivia App.

    1. cd front-end-react/
    2. npm install
    3. npm run start
    Catatan: Saat ini terdapat masalah pada webpack dependencies dengan versi Node v17. Jika Anda melihat error seperti ini: 'ERR_OSSL_EVP_UNSUPPORTED', pastikan untuk menjalankan aplikasi dengan versi Node v16.

  1. Setelah instalasi NPM dependencies beres, Anda mungkin akan melihat NPM warnings. Tenang, tak perlu dihiraukan untuk saat ini. Anda bisa lanjutkan ke langkah berikutnya.

  2. Pada AWS Cloud9 menu bar, pilih Previewâ†’Preview Running Applications. Anda akan melihat Trivia App berjalan di bagian kanan bawah. Silakan coba Trivia App sesuka hati.

  3. Terakhir, akhiri Node server yang sedang berjalan dengan menekan kombinasi Ctrl+C di AWS Cloud9 terminal dan tutup window pratinjau aplikasi.

Oke, aplikasi Trivia App sudah berjalan dengan baik. Namun, frontend masih berjalan di lokal (AWS Cloud9 environment). Sesuai rancangan awal, Kita perlu deploy frontend Trivia App ke Amazon S3 bucket.


Deploy Frontend ke S3 Bucket

Pada tahapan ini, Anda akan deploy frontend dari Trivia App ke Amazon S3 bucket.

  1. Pertama, buatlah S3 bucket dengan nama yang unik, misalnya menggunakan nama depan Anda, disusul dengan angka acak, lalu trivia-app-bucket. Contohnya seperti format berikut: <nama><angka>-trivia-app-bucket.

    1. aws s3 mb s3://<nama><angka>-trivia-app-bucket/
    Catatan: Jika nama S3 bucket yang Anda masukkan ternyata sudah ada orang lain yang pakai (already exist)), ubahlah bagian angka dengan yang lebih unik dan coba buat bucket lagi.

  1. Setelah itu, lakukan proses build untuk Trivia App dengan menjalankan perintah berikut di AWS Cloud9 terminal.

    1. npm run build
    Catatan: Setelah proses build, muncul sebuah folder bernama build yang berisi semua resource yang sudah teroptimasi dan bisa di-deploy.

  1. Setelah proses build usai, kini saatnya kita deploy frontend ke S3 bucket. Proses deploy ini sebenarnya adalah menyalin semua isi dari folder build ke S3 bucket yang barusan kita buat sekaligus membuka public access agar bisa diakses (read) secara publik.

    1. aws s3 sync --acl public-read build s3://<nama><angka>-trivia-app-bucket/

Mantap, frontend sudah berhasil di-deploy. Anda bisa akses aplikasi Trivia App melalui URL berikut: https://<nama><angka>-trivia-app-bucket.s3.amazonaws.com/index.html.


Menaruh Source Code di AWS CodeCommit

Tahapan terakhir, Anda akan membuat repository baru di AWS CodeCommit, lalu menggunakan git di AWS Cloud9, dan push (mengunggah/menyinkronkan) source code ke AWS CodeCommit repository.

  1. Klik logo AWS Cloud9 di paling kiri dari bagian menu bar dan pilih Go To Your Dashboard.

  2. Kemudian, akan terbuka browser tab baru yang membawa Anda ke halaman AWS Cloud9 Dashboard.

  3. Lalu, masuklah ke halaman AWS CodeCommit melalui fitur pencarian layanan.

  4. Klik tombol Create repository.

  5. Untuk Repository name, isikan trivia-app, kemudian pilih Create.

  6. Kembali ke browser tab yang memuat AWS Cloud9 (jangan tutup browser tab untuk AWS CodeCommit). Pada terminal, konfigurasikan git dengan nama dan email Anda.

    1. git config --global user.name "<GANTI_DENGAN_NAMA_ANDA>"
    2. git config --global user.email <GANTI_DENGAN_EMAIL_ANDA>

  1. Setelah itu, insialisasikan local repository Anda, navigasikan ke branch main, buat commit baru, lalu push source code Trivia App ke AWS CodeCommit repository.

    1. cd ~/environment/a428-cicd-labs/
    2. git init
    3. git checkout -b main
    4. git add .
    5. git commit -m "initial commit"
    6. git remote remove origin
    7. git remote add origin codecommit://trivia-app
    8. git push origin main
    Berikut penjelasan dari setiap perintah git di atas.

    • cd ~/environment/a428-cicd-labs/: Pindah ke direktori yang diinginkan.

    • git init: Menginisialisasi local git repository.

    • git checkout -b main: Bernavigasi atau pindah ke branch main.

    • git add .: Menambahkan semua berkas ke Git index. Kini semua berkas siap untuk di-commit.

    • git commit -m â€œinitial commitâ€: Membuat sebuah commit di Git log disertai dengan commit message.

    • git remote remove origin: Menghapus remote repository, yakni Dicoding Academy GitHub repository.

    • git remote add origin codecommit://trivia-app: Menambahkan remote repository baru (yakni AWS CodeCommit repository yang tadi dibuat) dengan nama origin. Perintah â€œremoteâ€ digunakan oleh git untuk melacak remote repository yang mana kini adalah CodeCommit repository. Prefiks â€œcodecommit://â€ memberi tahu git untuk menggunakan CodeCommit remote helper.

    • git push origin main: Mengunggah/menyinkronkan/memperbarui remote repository bernama origin (yakni codecommit://trivia-app) dengan main branch.

  1. Anda sudah berhasil push source code ke AWS CodeCommit repository. Mari kita lihat. Buka lagi browser tab yang memuat halaman AWS CodeCommit, klik menu Repositories yang ada di sebelah kiri, lalu pilih trivia-app.

  2. Semua kode yang kita miliki di AWS Cloud9 kini juga tersinkron di AWS CodeCommit. 

Hore! Anda sudah menuntaskan semua tahapan di latihan ini. Meski kita banyak bersinggungan dengan layanan-layanan AWS lain, tetapi sebenarnya fokus dari latihan ini adalah hanya pada bagian Code, yakni layanan AWS Cloud9 dan AWS CodeCommit.

Jangan khawatir bila Anda masih merasa kebingungan, apalagi soal AWS SAM (karena tools ini belum pernah dibahas di kelas-kelas yang kami sebutkan di bagian Prasyarat Kemampuan). Pada modul Continuous Deployment, kita akan membahas lebih lanjut soal AWS SAM dan membuat CI/CD Pipeline agar proses deployment lebih otomatis. Untuk saat ini, kita fokus saja ke continuous integration.

Jangan hapus dulu proyek Trivia App ini karena masih akan dipakai di beberapa latihan mendatang.
